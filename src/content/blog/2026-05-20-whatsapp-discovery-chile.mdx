El problema es simple pero doloroso: en Chile, el 40% de los laboratorios clínicos no tienen WhatsApp. Para Examya, esto significa que no podemos ofrecer nuestro servicio de procesamiento de órdenes médicas a nearly la mitad del mercado.

La solución tradicional sería contratar un equipo de investigadores que llame a cada laboratorio manualmente. O usar hojas de Excel con webscraping básico. Ambas opciones son lentas, costosas, y no escalan.

Decidimos construir un agente autónomo.

## El problema real

No es solo encontrar números de teléfono. Es cubrir el 100% de los laboratorios clínicos en Chile con información actualizada y precisa. En un país con 300+ laboratorios, esto significa:

- Verificar si cada laboratorio tiene WhatsApp
- Encontrar el número correcto cuando está oculto
- Actualizar números que cambian
- Hacer esto continuamente, no solo una vez

La escala humana no basta. Necesitamos automatización inteligente.

## Arquitectura del sistema

El corazón de WhatsApp Discovery es un sistema modular que rota estrategias automáticamente. Cada "ronda" de scraping procesa 10 laboratorios y prueba diferentes enfoques.

### Estrategia 1: Consulta avanzada de base de datos

Primero, consultamos nuestra base de datos interna con queries complejos:

```sql
SELECT l.nombre, l.web, l.region, 
       CASE WHEN l.whatsapp IS NOT NULL AND l.whatsapp != '' 
            THEN l.whatsapp 
            ELSE NULL END as whatsapp_directo
FROM laboratorios l
WHERE l.whatsapp IS NULL 
   OR l.whatsapp = ''
   OR l.whatsapp_actualizado < NOW() - INTERVAL '3 months'
   AND l.region IN ('LOS LAGOS', 'LOS RÍOS', 'ARICA Y PARINACOTA')
LIMIT 10;
```

Este filtro nos da laboratorios específicos que necesitan atención prioritaria.

### Estrategia 2: Búsqueda directa con Google Search API

Cuando la base de datos no tiene información, usamos Google Search API con queries precisos:

```javascript
const searchQuery = `"${laboratorio.nombre}" WhatsApp número teléfono Chile ${laboratorio.region}`;
```

Buscamos patrones específicos como:
- "Laboratorio XYZ WhatsApp"
- "Contacto Laboratorio XYZ teléfono"
- "WhatsApp +56 ${nombre}"

### Estrategia 3: Scraping de Instagram Bio Bypass

Muchos laboratorios ocultan sus números de WhatsApp en Instagram. Aquí usamos técnicas avanzadas:

1. Extraemos el perfil de Instagram del laboratorio
2. Buscamos en el biografía menciones de WhatsApp
3. Usamos expresiones regulares para encontrar números:
   ```regex
   /(?:whatsapp|wa\.me|WhatsApp)\s*(?::|:|)\s*(?:\+569|569|9)\d{7}/gi
   ```

### Estrategia 4: Descubrimiento Multiplataforma

Combinamos información de múltiples fuentes:
- Páginas web del laboratorio
- Facebook Business Pages
- LinkedIn
- Directorios médicos online
- Mapas de Google

Cada fuente nos da diferentes piezas del rompecabezas.

## Rotación de estrategias

La clave está en la rotación automática. Si una estrategia falla 3 veces para una región, el sistema automáticamente cambia a otra. Esto mantiene la efectividad alta.

El patrón es simple:

```
Ronda 127: Database Query + Google Search + Instagram Bio
Ronda 128: Enhanced Multi-Platform + Instagram Direct Scraping  
Ronda 129: Database Query + Google Search + Instagram Bio (retry)
```

Cada ronda procesa 10 laboratorios y reporta resultados en tiempo real.

## Manejo de errores robusto

El mundo real es caótico. Las páginas web caen, los APIs tienen rate limits, los números cambian. Nuestro sistema debe ser resiliente.

```javascript
try {
  const resultado = await scrapLaboratorioWhatsApp(laboratorio);
  if (resultado.whatsapp) {
    await guardarNumeroWhatsApp(laboratorio.id, resultado.whatsapp);
    return { success: true, whatsapp: resultado.whatsapp };
  }
} catch (error) {
  logger.error(`Error scraping ${laboratorio.nombre}:`, error.message);
  // El sistema automáticamente reintenta en la próxima ronda
}
```

Si falla, el laboratorio se vuelve a procesar en la siguiente ronda con una estrategia diferente.

## Lo que aprendí

### 1. La web es más compleja de lo que parece

Cada laboratorio tiene su propio formato. Algunos ponen WhatsApp en la esquina derecha, otros en el footer, otros lo ocultan detrás de formularios. No hay una solución única.

### 2. La rotación es clave

Ninguna estrategia funciona al 100%. La combinación de múltiples enfoques con rotación automática es lo que mantiene el sistema efectivo.

### 3. La calidad importa más que la cantidad

Encontrar 100 números falsos no sirve. Validamos cada número con pruebas automáticas de formato y existencia.

### 4. La automatización debe ser inteligente

No se trata de scrapear todo sin parar. Es sobre ser estratégico: qué laboratorios prioritarios, qué estrategias probar, cuándo reintentar.

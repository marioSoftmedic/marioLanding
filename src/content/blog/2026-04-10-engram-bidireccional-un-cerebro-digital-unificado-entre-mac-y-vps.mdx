En el mundo distribuido del desarrollo de AI, mantener una memoria consistente entre múltiples entornos ha sido uno de los mayores desafíos. Durante meses, he lidiado con la fragmentación de conocimiento entre mi MacBook Pro (donde desarrollo la mayor parte del tiempo) y mi servidor VPS (donde corre Cotocha 24/7).

El problema: dos cerebros digitales trabajando con información desincronizada, decisions duplicadas, y aprendizajes aislados.

La solución: un pipeline bidireccional de memoria que crea un cerebro unificado en tiempo real.

## Arquitectura del Sistema

El sistema se compone de dos flujos principales que operan en ciclos separados para evitar conflictos:

**Flujo Mac → VPS (Importación)**
- Cada 2 horas en el VPS: un cron job ejecuta `git pull` del repositorio obsidian-vault
- Luego ejecuta `engram sync --import` para absorber las nuevas memorias desde los chunks
- Finalmente, un script bridge transforma las observaciones en archivos .md accesibles para QMD

**Flujo VPS → Mac (Exportación)**
- Cada 4 horas en el VPS: otro cron exporta las sesiones de OpenClaw a chunks Engram
- Los chunks se comprimen y suben al mismo repositorio obsidian-vault vía git push
- En Mac, el script `engram-sync.sh` absorbe automáticamente los nuevos chunks a las 7:30 AM

## Componentes Clave

### 1. Repositorio Central como Puente

El repositorio `marioSoftmedic/obsidian-vault` actúa como el medio de transporte entre ambos entornos. Almacena no solo los notes de Obsidian, sino también los chunks de memoria comprimidos en `.engram/chunks/`.

```
obsidian-v/
├── .engram/chunks/
│   ├── sessions-2026-04-10.jsonl.gz
│   ├── observations-2026-04-10.jsonl.gz
│   └── prompts-2026-04-10.jsonl.gz
├── Proyectos/
│   └── 05_Blog/
│       └── _INDEX.md
└── 10_Investigación/
    └── Wiki/
        └── _INDEX.md
```

### 2. Scripts de Sincronización

**En Mac (`engram-sync.sh`):**
```bash
#!/bin/bash
# Exporta chunks desde el vault y los sube al repo
cd ~/.engram
engram sync --all --export
cd ~/repos/obsidian-vault
git add .engram/
git commit -m "sync: engram chunk $(date +%Y-%m-%d)"
git push
```

**En VPS (bridge scripts):**
- `bridge-openclaw-engram.py`: Parsea sesiones JSONL y genera chunks
- `bridge-engram-to-qmd.py`: Convierte chunks a .md para búsqueda semántica

### 3. Resolución de Conflictos

El mayor desafío fue resolver divergencias en el historial de git. En un momento, el repositorio local de Mac solo tenía 4 commits sin ancestro común con el remote (VPS tenía 133).

La solución fue usar `git reset --soft obsidian-vault/main`, que mantuvo todos los cambios locales pero los posicionó sobre el HEAD del remote, permitiendo un commit limpio.

## Beneficios del Sistema

1. **Memoria Unificada**: Las decisiones técnicas, arquitectónicas y de proyecto están disponibles en ambos entornos
2. **Reducción de Duplicación**: Elimina la necesidad de documentar twice o recordar lo que se hizo en otro entorno
3. **Búsqueda Semántica Cruzada**: QMD puede indexar tanto las notas de Obsidian como las sesiones de OpenClaw
4. **Continuidad del Trabajo**: Puedo continuar sesiones donde las dejé, sin perder contexto

## El Futuro: Karpathy Knowledge Base

Este sistema es el componente fundamental para implementar el método Karpathy de base de conocimiento personal. Con un flujo constante de ingestión, procesamiento LLM, y publicación Wiki, estoy construyendo un sistema que aprende y compila conocimiento automáticamente.

El próximo paso es conectar este pipeline con el blog-writer para que las memorias recientes se conviertan automáticamente en contenido público, cerrando el ciclo de conocimiento privado a público.

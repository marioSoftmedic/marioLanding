El problema era simple pero complejo. En un país con miles de laboratorios clínicos, encontrar prospectos calificados para una plataforma médica como Examya requería verificar manuales cada teléfono, sitio web y presencia digital. Una labor que tomaba semanas y generaba datos inconsistentes.

Lo que construí fue un sistema de prospección automatizada. Un agente autónomo que viaja por Chile digitalmente, identifica clínicas, valida información y califica prospectos. Todo a través de una API de WhatsApp que convierte conversaciones en acciones.

## Cómo funciona

El sistema tiene tres componentes principales: discovery, validación y calificación.

### Discovery Engine

El corazón está en un crawler que busca prospectos. Usamos Puppeteer para navegar sitios web y extraer datos estructurados.

```typescript
interface Prospect {
  name: string;
  phone?: string;
  website?: string;
  specialties: string[];
  location: {
    region: string;
    city: string;
    address?: string;
  };
  status: 'pending' | 'contacted' | 'qualified' | 'rejected';
}

class ClinicDiscoveryAgent {
  async discoverInRegion(region: string): Promise<Prospect[]> {
    const searchResults = await this.webSearch(region, 'laboratorio clínico');
    const prospects: Prospect[] = [];
    
    for (const result of searchResults) {
      const prospect = await this.analyzeWebsite(result.url, region);
      if (prospect) {
        prospects.push(prospect);
      }
    }
    
    return prospects;
  }

  private async analyzeWebsite(url: string, region: string): Promise<Prospect | null> {
    try {
      const browser = await puppeteer.launch({ headless: true });
      const page = await browser.newPage();
      
      await page.goto(url, { waitUntil: 'networkidle2' });
      const content = await page.content();
      
      const $ = cheerio.load(content);
      
      const name = $('h1, .business-name, .clinic-name').first().text().trim();
      const phone = this.extractPhone(content);
      const specialties = this.extractSpecialties(content);
      
      await browser.close();
      
      return {
        name,
        phone,
        website: url,
        specialties,
        location: { region, city: this.extractCity(content) },
        status: 'pending'
      };
    } catch (error) {
      console.error(`Failed to analyze ${url}:`, error);
      return null;
    }
  }
}
```

### Validación via WhatsApp

La magia ocurre cuando el sistema contacta prospectos a través de WhatsApp. Usamos el flujo de conversación natural para validar información.

```typescript
class WhatsAppProspectValidator {
  async validateProspect(prospect: Prospect): Promise<Prospect> {
    const contactResponse = await this.sendWhatsAppMessage(
      prospect.phone!,
      'Hola, soy de Examya, ¿podrían confirmar si realizan exámenes de laboratorio?'
    );
    
    if (contactResponse.status === 'delivered') {
      const response = await this.waitForResponse(prospect.phone!);
      
      if (response.contains('laboratorio') || response.contains('exámenes')) {
        prospect.status = 'qualified';
        prospect.specialties = this.extractSpecialtiesFromResponse(response);
      } else {
        prospect.status = 'rejected';
      }
    } else {
      prospect.status = 'rejected';
    }
    
    return prospect;
  }
}
```

### Sistema de Calificación

Cada prospecto recibe una puntuación basada en criterios objetivos. Usamos un sistema ponderado para identificar los mejores leads.

```typescript
class ProspectScorer {
  calculateScore(prospect: Prospect): number {
    let score = 0;
    
    // Puntuación por teléfono válido
    if (prospect.phone) score += 30;
    
    // Puntuación por especialidades relevantes
    const relevantSpecialties = ['hematología', 'bioquímica', 'microbiología'];
    const matchCount = prospect.specialties.filter(s => 
      relevantSpecialties.some(rs => s.toLowerCase().includes(rs))
    ).length;
    score += matchCount * 20;
    
    // Puntuación por región (prioridad en áreas específicas)
    const priorityRegions = ['Metropolitana', 'Valparaíso', 'Biobío'];
    if (priorityRegions.includes(prospect.location.region)) {
      score += 25;
    }
    
    return Math.min(score, 100);
  }
}
```

## Lo que aprendí

El primer gran desafío fue la calidad de los datos. Muchos sitios web de clínicas tienen información desactualizada o incompleta. Tuvimos que implementar un sistema de verificación en múltiples pasos:

1. **Búsqueda semántica**: Usamos embeddings para entender el contexto de los sitios web
2. **Validación cruzada**: Cruzamos información entre sitio web, redes sociales y directorios médicos
3. **Flujo de conversación**: La interacción humana confirmaba la información

El segundo desafío fue el cumplimiento normativo. En salud, la privacidad es crítica. Implementamos:

- Encriptación de todos los datos sensibles
- Consentimiento explícito antes de contactar
- Retención de datos limitada a 30 días
- Auditoría completa de todas las interacciones

El sistema procesa ahora prospectos 10 veces más rápido que el método manual, con un tasa de calificación del 78%. Los datos son consistentes y actualizados semanalmente.

API Legacy vs Arquitectura FHIR + Ley 21.719

Por qué "tener una API" ya no es suficiente para la interoperabilidad legal

La trampa de "Tener API"

  • • Proveedores legacy exportan su base de datos como un JSON plano sin estándar clínico.
  • • Mueven datos sin registrar el propósito (violando el Accountability de la Ley 21.719).
  • • Obligan a la clínica a costear y programar las transformaciones para conectarse al MINSAL.

El estándar FHIR + Compliance

  • • Cada dato viaja estructurado: DiagnosticReport, ServiceRequest, Patient.
  • • Fhirex actúa de middleware: intercepta la API legacy, la valida, y registra un AuditEvent (RAT).
  • • Protege al directorio de la clínica frente a multas de hasta 1.500 millones de pesos.
